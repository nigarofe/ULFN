# Sequence Diagrams
```mermaid
sequenceDiagram
    autonumber
    actor User
    participant Frontend
    participant Backend
    participant Database

    User->>Frontend: Submit Login Form
    activate Frontend
    Frontend->>Backend: POST /login
    activate Backend
    
    Backend->>Database: Query User Credentials
    activate Database
    Database-->>Backend: Return Hash
    deactivate Database
    
    alt Password Match
        Backend-->>Frontend: 200 OK + JWT
        Frontend-->>User: Redirect to Dashboard
    else Password Mismatch
        Backend-->>Frontend: 401 Unauthorized
        Frontend-->>User: Show Error Message
    end
    
    deactivate Backend
    deactivate Frontend
    
    opt Analytics
        Frontend-)Analytics: Log User Login Attempt
    end
```

# State Diagrams
```mermaid
stateDiagram-v2
    [*] --> Off
    
    state Off {
        [*] --> PowerDisconnected
    }
    
    Off --> On : Power Button Pressed
    
    state On {
        [*] --> Booting
        Booting --> Running : OS Loaded
        
        state Running {
            state fork_state <<fork>>
            [*] --> fork_state
            fork_state --> Processing
            fork_state --> Monitoring
            
            Processing --> Done : Task Complete
            Monitoring --> Alert : Error Detected
            
            state join_state <<join>>
            Done --> join_state
            Alert --> join_state
        }
        Running --> Standby : Idle 10 mins
        Standby --> Running : Activity Detected
    }
    
    On --> Off : Shutdown Initiated
```

# Entity Relationship Diagrams (ERD)
```mermaid
erDiagram
    CUSTOMER ||--o{ ORDER : places
    CUSTOMER {
        int id PK
        string name
        string email
    }
    ORDER ||--|{ LINE-ITEM : contains
    ORDER {
        int order_number PK
        date order_date
        int customer_id FK
    }
    PRODUCT ||--o{ LINE-ITEM : "appears in"
    PRODUCT {
        string product_code PK
        string description
        float price
    }
    LINE-ITEM {
        int order_number PK, FK
        string product_code PK, FK
        int quantity
    }
```
